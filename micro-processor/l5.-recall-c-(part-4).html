Stack

Stack

Stack은 데이터를 저장하기 위한 간단한 structure로 사용되는데,

  1. Data가 도착한 순서가 중요하다.
  2. 데이터가 생성되고 없어지는 것 모두 TOP에서 이뤄진다.
  3. Last in First out (LIFO)

Terminologies & Operation

Under flow

텅 빈 stack에서 Pop하려는 것

Overflow

꽉 찬 stack에 Push하려는 것

Operation

Push

Stack에 element를 삽입하는 과정이다.

POP

Stack에서 element를 제거하는 과정이다.

Top

Top의 데이터를 확인한다.

IsEmptyStack() & IsFullStack()

Stack에 데이터가 비었는가 꽉 찼는가? Overflow와 Underflow 방지

Implementation

Simple Array Based Implemenatation

Array기반으로 구현할 때에는 head의 위치를 element가 있는 곳 중 제일 index가 큰 곳으로 지정한다.

Time Complexity of IsFullStackO(1)

Disadvantage

  1. Stack의 Maximum size가 정해져 있어야 한다.
  2. Size가 변경될 수 없다.

Array vs Linked list

Array

  1. operation이 당연하게도 constant time.
  2. Doubling해서 allocation하는 데에 한 번에 조금 공간 점유가 크다.
  3. n개의 operation이 들어오면 O(n)만큼 걸린다.

Linked List

  1. Grows와 Shrink모두 쉽게 Gracefully 구현 가능하다.
  2. 모든 operation이 O(1) 만큼의 시간이 걸린다.
  3. 모든 operation은 reference를 위해 추가적인 공간과 시간을 사용한다.

Queue

Queue

Queue는 Insertion이 rear에서 이뤄지고, deletion이 front에서 이뤄진다.

Terminologies & Operation

Enqueue, Dequeue

Enqueue:

  1. queue에 element를 insertion하는 과정
  2. 꽉 찬 queue에 Enqueue 하려고 하면 Overflow

Dequeue:

  1. queue에서 element를 제거하는 과정
  2. 텅 빈 Queue에서 Dequeue 하려고 하면 Underflow

Operation

Front

가장 앞에 있는 element를 제거 없이 가져온다.

Queue Size

Queue에 저장된 element의 수를 출력한다, Overflow 방지 가능

isEmptyQueue

비어있는 Queue이지 판별하는 함수, Under flow 방지 가능

Applications

별건 없고 잠깐 동안 대기 해야 하는 대기 열 같은 걸 큐로 말한다.
-> 미국에서도 Is this a line for ~ 하는 거보다 Is this a queue for ~? 하는 걸 더 잘 알아 듣더라

Implementatioin

Simple Circular array based Implementation

Circular queue를 이용하는 이유

Circular array를 사용하는 이유는 queue에서 rear를 새로 할당하고 front를 버리는 과정을 이용하게 되면, 계속해서 front 부분에서 initial slot이 낭비되는 문제가 생긴다.

Enqueue & Dequeue

  1. Enqueue와 Dequeue를 못하는 상황에서 Exception을 throw한다.
  2. Front 와 Rear가 -1을 가리키면 queue가 비었다는 것이다.

Priority Queue

definition

Ascending-priority: 가장 작은 키를 가진 요소가 높은 우선도를 가진다.
Descending-priority: 가장 큰 키를 가진 요소가 높은 우선도를 가진다.

Operation

Insert(key, data)

key value에 따라서, 요소의 oreder를 정하여 priority queue에 삽입한다.

DeleteMin()/DeleteMax()

가장 작은/가장 큰 key의 요소를 반환하면서 지운다.

GetMinimum()/GetMaximum()

가장 작은/ 가장 큰 key의 요소를 반환한다. with out deleting

Auxiliary

k-th smallest/largest : k-th 작은/ 큰 원소를 반환한다.
Size: priority queue의 element 개수를 출력한다.
