# 서울 브레멘 사이트 설정 가이드 🎵

이 사이트는 **서버 없이** 구글 시트/드라이브와 연동됩니다.
주소: `https://minhjih.github.io/seoulbremen/`

- **합주 일정 · 연습곡 · 멤버** → 구글 시트에 저장
- **사진** → 구글 드라이브 **공유 폴더 1개**에 모음. 사이트에서 누구나 업로드, 폴더 안 사진은 자동으로 갤러리에 표시
- **웹에서 추가/수정/삭제** → 구글 앱스 스크립트를 통해 시트·드라이브에 바로 반영

처음에는 데모 데이터(`data.json`)로 동작하고, 아래 설정을 마치면 구글 시트와 연결됩니다.

---

## 1단계. 구글 시트 만들기

1. [sheets.google.com](https://sheets.google.com) 에서 새 시트를 만듭니다.
2. 아래 **3개의 탭(시트)** 을 만들고 이름을 *정확히* 맞춥니다 (소문자):
   `rehearsals`, `songs`, `members`
   (사진은 시트가 아니라 드라이브 폴더로 관리합니다 — 3단계 참고)
3. 각 탭의 **1번째 줄(머리글)** 을 아래처럼 입력합니다. (열 순서는 자유, 이름만 맞으면 됨)

### `rehearsals` (합주 일정)
| date | time | location | address | songs | attendees | cost | notes |
|------|------|----------|---------|-------|-----------|------|-------|
| 2026-06-14 | 14:00-15:00, 15:00-16:00 | 낙원상가 합주실 A룸 | 서울 종로구 삼일대로 428 | 첫 곡, 사운드체크 | 최진형, 홍길동 | 40000 | 악기 챙겨오기 |

- `songs`, `attendees` 는 여러 개일 때 **쉼표(,)** 로 구분합니다.
- `time` 은 1시간 단위 시간대를 **쉼표(,)** 로 나열합니다. 사이트 편집 화면에서는 체크박스로 여러 시간을 고르면 되고, 연속된 시간은 자동으로 합쳐서 표시됩니다 (예: `14:00-15:00, 15:00-16:00` → `14:00 - 16:00`).
- `address` 를 넣으면 사이트에서 클릭 시 네이버 지도로 연결됩니다.
- `cost` 는 합주실 총비용(숫자)을 넣으면 **참석자 수로 나눈 1인당 정산액**이 자동 표시됩니다.

### `songs` (연습곡)
| title | artist | status | key | link | notes |
|-------|--------|--------|-----|------|-------|
| 예시 곡 | 아티스트 | 연습중 | C | https://youtu.be/... | 인트로 다듬기 |

- `status` 는 `후보` / `연습중` / `완성` 중 하나를 쓰면 색이 구분됩니다.

> 📷 **사진은 시트 탭이 필요 없습니다.** 드라이브 폴더로 관리합니다 (3단계 참고).

### `members` (멤버)
| name | part | joined |
|------|------|--------|
| 최진형 | 리더 / 기타 | 2026 |

## 2단계. 앱스 스크립트로 "웹에서 수정" 기능 켜기 (추천)

이 단계를 해야 사이트에서 직접 추가·수정·삭제하고, 사진을 업로드할 수 있습니다.

1. 구글 시트 상단 메뉴 **[확장 프로그램] → [Apps Script]** 를 엽니다.
2. 편집기의 기존 코드를 모두 지우고, 이 폴더의 **`apps-script.gs`** 내용을 전부 붙여넣습니다.
3. 맨 위의 값들을 채웁니다:
   - `var EDIT_KEY = "change-me";` → `"change-me"` 를 원하는 **관리자 편집 비밀번호**로 변경
   - `var UPLOAD_KEY = "bremen0101";` → 원하는 **사진 업로드 비밀번호** (config.js 와 동일하게)
   - `var PHOTOS_FOLDER_ID = "";` → **3단계**에서 만든 사진 폴더 ID 붙여넣기
4. 우측 상단 **[배포] → [새 배포]** 클릭 → 톱니바퀴 → **"웹 앱"** 선택
   - **설명**: 아무거나
   - **실행 계정**: 나
   - **액세스 권한**: **모든 사용자**
5. **[배포]** 를 누르고 권한을 허용하면 **웹앱 URL** 이 나옵니다 (`.../exec` 로 끝남). 복사해 둡니다.

> 나중에 `apps-script.gs` 를 수정하면 **[배포] → [배포 관리] → 연필(편집) → 버전: 새 버전 → 배포** 로 다시 배포해야 반영됩니다. (URL은 그대로 유지됩니다)

## 3단계. 사진 공유 폴더 1개 만들기

1. 구글 드라이브에서 **새 폴더**를 하나 만듭니다 (예: "서울 브레멘 사진").
2. 폴더를 우클릭 → **[공유] → [공유]** → 일반 액세스를
   **"링크가 있는 모든 사용자 - 뷰어"** 로 바꿉니다. (중요! 안 하면 사진이 안 보임)
3. 폴더를 열면 주소가 `https://drive.google.com/drive/folders/<폴더ID>` 입니다.
   가운데 **폴더 ID** 를 복사해 2단계의 `PHOTOS_FOLDER_ID` 에 붙여넣습니다.
   (원하면 `config.js` 의 `DRIVE_FOLDER_ID` 에도 넣으면 사이트에 "📁 드라이브 폴더" 바로가기가 생깁니다.)

이제 사진은 두 가지 방법으로 올릴 수 있고, **폴더에 들어오면 자동으로 갤러리에 표시**됩니다.

- **사이트에서**: 갤러리의 **"📸 사진 올리기"** 버튼 → 업로드 비밀번호(`UPLOAD_KEY`) 입력 후 폰/PC에서 바로 업로드 (구글 계정 불필요)
- **드라이브에서**: 이 폴더에 사진을 직접 드래그해서 올려도 됩니다.

> 업로드 비밀번호는 동아리원끼리 공유하는 용도이며, 합주·멤버를 편집하는 관리자 비밀번호(`EDIT_KEY`)와는 별개입니다.

> 업로드한 사진은 자동으로 적당한 크기로 줄여 저장되어 용량을 아끼고 빠르게 표시됩니다.
> 관리(편집) 모드에서는 각 사진의 🗑️ 버튼으로 삭제할 수 있습니다 (드라이브 휴지통으로 이동).

## 4단계. 사이트에 연결하기 (`config.js` 수정)

`seoulbremen/config.js` 파일을 열어 아래 값을 채웁니다.

```js
window.SEOUL_BREMEN_CONFIG = {
  SCRIPT_URL: "https://script.google.com/macros/s/AKfy.../exec", // 2단계의 웹앱 URL
  EDIT_KEY: "내가-정한-비밀번호",   // 2단계에서 정한 EDIT_KEY 와 똑같이!
  UPLOAD_KEY: "bremen0101",         // 사진 업로드 비밀번호 (앱스 스크립트 UPLOAD_KEY 와 똑같이!)
  DRIVE_FOLDER_ID: "",              // (선택) 3단계 사진 폴더 ID — "드라이브 폴더" 바로가기 표시용
  SHEET_ID: "",                     // 앱스 스크립트를 쓰면 비워둬도 됨
  TABS: { rehearsals: "rehearsals", songs: "songs", photos: "photos", members: "members" },
};
```

저장하고 깃허브에 푸시(또는 커밋)하면 끝입니다.

## 사용법

- 사이트 우측 상단 **[관리]** 클릭 → 비밀번호 입력 → 편집 모드 진입
- 각 섹션의 **[+ 추가]** 버튼, 각 항목의 **✏️ 수정 / 🗑️ 삭제** 버튼으로 관리
- 변경 내용은 구글 시트에 바로 저장됩니다. 휴대폰에서도 가능합니다.

## 자주 묻는 질문

- **사진이 안 보여요** → 사진 폴더 공유 설정이 "링크가 있는 모든 사용자 - 뷰어"인지, `PHOTOS_FOLDER_ID` 가 맞는지 확인하세요.
- **사진 업로드가 안 돼요** → 앱스 스크립트의 `PHOTOS_FOLDER_ID` 가 채워져 있고, 코드 수정 후 "새 버전"으로 다시 배포했는지 확인하세요.
- **저장이 안 돼요 / 비밀번호 오류** → `config.js`의 `EDIT_KEY` 와 `apps-script.gs`의 `EDIT_KEY` 가 같은지, 웹앱 배포 권한이 "모든 사용자"인지 확인하세요.
- **시트만 쓰고 웹 편집은 안 할래요** → 2단계를 건너뛰고, 시트를 [파일]→[공유]→[웹에 게시] 한 뒤 `config.js`의 `SHEET_ID` 만 채우면 읽기 전용으로 동작합니다.

## 파일 구조

```
seoulbremen/
├─ index.html       사이트 페이지
├─ styles.css       디자인
├─ app.js           데이터 로딩/렌더링/편집 로직
├─ config.js        ← 여기에 시트/스크립트 정보 입력
├─ data.json        데모(기본) 데이터
├─ apps-script.gs   구글 시트에 붙여넣을 백엔드 코드
└─ photos/          (선택) 직접 올릴 사진 폴더
```
